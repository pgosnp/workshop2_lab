Lab 3 - Rapid Web Development with Sails.js

Sails.js intends to mimic Ruby on Rails, which allows rapid web application development under MVC (Model-View-Controller) framework.

Software installation

Make sure Node.js has been installed in your station, as Sails runs on top of Node.js.

Bring up the Terminal and install sails.js with these commands

su xdeveloper

sudo npm install -g sails

exit

Creating a new project

After everything have been installed, we can create a new project by running

sails new lab3

A project folder is generated. Please go into this folder and generate a pair of controller and model, to be named as Person.

cd lab3

sails generate api Person

Two files (PersonController.js and Person.js) were generated under different folders. Open the entire project with VS Code.

Finally, let's start the server in the terminal with command

sails lift

You may be given 3 options. Pick 1 (safe) to keep all existing data and option 3 (drop) will remove all the existing data. Option 2 (alter) will try to keep all existing data while make the necessary changes to fit any new database setting. Let's use option 2 for today's lab. We may also simply specify this setting with the following sails lift command

sails lift --models.migrate='alter'

Now, visit http://localhost:1337/ with your browser to see the default page of this site..

Creating a new item

Add the following function to your PersonController.js, inside the braces.

// create function
create: function (req, res) {
    if (req.method == "POST") {
        Person.create(req.body.Person).exec(function (err, model) {
            return res.send("Successfully Created!");
        });
    } else {
        return res.view('person/create');
    }
},

Under the views folder, create a new folder named person. Create a new file named as create.ejs.

<!--create.ejs-->
<form action="/person/create" method="POST">
    <div class="form-group">
        <label>Name</label>
        <input type="text" class="form-control" name="Person[name]">
    </div>
    <div class="form-group">
        <label>Age</label>
        <input type="text" class="form-control" name="Person[age]">
    </div>
    <button type="submit" class="btn btn-primary">Submit</button>
</form>

Now, restart the Sails engine and visit http://localhost:1337/person/create. The form should be displayed.

Generating a JSON output

One way to view the stored items in our database is to generate a JSON feed. Add the following in the PersonController.js again.

// json function
json: function (req, res) {
    Person.find().exec(function (err, persons) {
        return res.json(persons);
    });
},

That's it. Now, restart the server and visit http://localhost:1337/person/json to see your inputted data.

Retrieve all items

Similarly, we can display all saved items on a webpage. Let's add the following function to the PersonController.js again.

// index function
index: function (req, res) {
    Person.find().exec(function (err, persons) {
        return res.view('person/index', { 'persons': persons });
    });
},

Then, we have to provide the corresponding index.ejs file. Create this file under the person folder.

<!--index.ejs-->
<table class="table">
    <% persons.forEach( function(model) { %>
        <tr><td><%= model.name %></td><td><%= model.age %></td></tr>
    <% }); %>
</table>

Under Sails.js, if no controller function name is provided in the url, the index function will be called. Now, let's simply visit http://localhost:1337/person/ to see all the stored items.

Viewing a single item

As we want to view a single item here, we have to use findOne() instead of find(). findOne() would return the first matching item.

// view function
view: function (req, res) {
    Person.findOne(req.params.id).exec(function (err, model) {
        if (model != null)
            return res.view('person/view', { 'person': model });
        else
            return res.send("No such person");
    });
},

Then, add a new view.ejs under the /views/person folder. Please create this file yourself to display the selected item.

This page could be reached via http://localhost:1337/person/view/1. Here, 1 is the primary key id of the Person model. In the Person controller, we can obtain this value via req.params.id.

Deleting an item

￼￼￼￼￼ Create another function in the controller, as follows

// delete function
delete: function(req, res) {
   Person.findOne(req.params.id).exec( function(err, model) {
        if (model != null) {
            model.destroy();
            return res.send("Person Deleted");
        } else {
            return res.send("Person not found");
        }
    });
},

To delete an item, first we have to find it from our Person model. Then we issue the destroy() method to delete that particular item. Visit http://localhost:1337/person/delete/1 to remove the first item in your Person model.

Please be reminded that it is not a good idea to delete using the GET method. The above only serves as a simplified demonstration.

Updating an item

Update is more difficult, as it involves two functions:

  1. Show a form with filled data;
  2. Modify the stored values based on the form input. These two functions are using different HTTP request methods, and thus could be seperatedly handled in the following controller function.
// update function
update: function (req, res) {
    if (req.method == "GET") {
        Person.findOne(req.params.id).exec(function (err, model) {
            if (model == null)
                return res.send("No such person!");
            else
                return res.view('person/update', { 'person': model });
        });
    } else {
        Person.findOne(req.params.id).exec(function (err, model) {
            model.name = req.body.Person.name;
            model.age = req.body.Person.age;
            model.save();
            return res.send("Record updated");
        });
    }
},

Also provide the update.ejs to display the form as filled with existing data.

<!--update.ejs-->
<form action="/person/update/<%= person.id %>" method="POST">
    <div class="form-group">
        <label>Name</label>
        <input type="text" class="form-control" name="Person[name]" value="<%= person.name %>">
    </div>
    <div class="form-group">
        <label>Age</label>
        <input type="text" class="form-control" name="Person[age]" value="<%= person.age %>">
    </div>
    <button type="submit" class="btn btn-primary">Submit</button>
</form>

Searching

Here, we provide a search function. Users can search items based on name, age or even both. 

// search function
search: function (req, res) {

    const qName = req.query.name || "";
    const qAge = req.query.age || "";

    if (qAge == "") {

        Person.find()
            .where({ name: { contains: qName } })
            .sort('name')
            .exec(function (err, persons) {
                return res.view('person/index', { 'persons': persons });
            })

    } else {
        Person.find()
            .where({ name: { contains: qName } })
            .where({ age: qAge })
            .sort('name')
            .exec(function (err, persons) {
                return res.view('person/index', { 'persons': persons });
            })
    }
},

The search queries are submitted with the query strings. Since query strings are easily editable in the URL, there's a chance that these query strings are not provided. In this way, either req.query.name or req.query.age could become undefined. In order to avoid this, line 4 and 5 make use of the coalesing operators || to provide some default values to our constant variable qName and qAge.

Notice the use of contains in line 10 and 18, which allows partial matching. Omitting this keyword will perform exact matching.

Now, let's test this controller function with http://localhost:1337/person/search? together with query string like name=martin&age=26. The search form is not provided, I'd like to leave it as an exercise for you.

More examples on the database query: https://sailsjs.com/documentation/concepts/models-and-orm/query-language

Pagination

When we have many records in our database, a lot of time we don't want to display all them in one page. We can easily handle this with pagination. Now, let's add the following function in your controller again.

// paginate function
paginate: function (req, res) {

    const qPage = req.query.page || 1;

    Person.find().paginate({ page: qPage, limit: 2 }).exec(function (err, persons) {
        Person.count().exec(function (err, value) {
            var pages = Math.ceil(value / 2);
            return res.view('person/paginate', { 'persons': persons, 'count': pages });
        });
    });
},

The Waterline ORM engine provides another handy method for us. paginate() would return those items in a particular page (to be specified by page: ) and the number of items in a page could be defined with limit:. Another method count() would return the number of items in the Person model. We divide this number by 2 as each page shows two items.

Then, here's the paginate.ejs.

<!--paginate.ejs-->
<table class="table">
    <% persons.forEach( function(model) { %>
        <tr><td><%= model.name %></td><td><%= model.age %></td></tr>
    <% }); %>
</table>

<% const current = Number(req.query.page || 1); %>

<nav aria-label="Page navigation example">
    <ul class="pagination">
        <li class="page-item">
            <a class="page-link" href="/person/paginate/?page=<%= Math.max(current-1, 1) %>">Previous</a>
        </li>

        <% for (i = 1; i <= count; i++) { %>
            <li class="page-item">
                <a class="page-link" href="/person/paginate/?page=<%= i %>"><%= i %></a>
            </li>
        <% } %>

        <li class="page-item">
            <a class="page-link" href="/person/paginate/?page=<%= Math.min(current+1, count) %>">Next</a>
        </li>
    </ul>
</nav>

Line 2 to 6 display the paginated items with a HTML table. Line 10 to 26 display the pagination links. Since the req object is available in the ejs file, in line 8, we access the req.query.page value, which represents the current page number. If it is not provided, we assume the first page should be returned.

Playing with routes

Up till now, our index page could be reached via http://localhost:1337/person/. If we want it to be our default page of our site, we can set it via routes.js, also located under config folder.

// '/': {
//   view: 'homepage'
// }

'/' : 'PersonController.index'

Comment out the existing lines, which display homepage.ejs by default. With the above change, when the user visits http://localhost:1337/, the server will execute the index function inside the Person controller, thus showing our first page.
