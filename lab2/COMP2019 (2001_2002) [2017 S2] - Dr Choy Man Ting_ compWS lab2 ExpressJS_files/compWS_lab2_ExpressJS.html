Lab 2: Basic CRUD with Express.js

Today, we are going to develop our first web application using the MEAN stack. MEAN is a free and open-source JavaScript software stack for building dynamic web sites and web applications.

The MEAN stack is MongoDB, Express.js, Angular.js, and Node.js. Because all components of the MEAN stack support programs are written in JavaScript, MEAN applications can be written in one language for both server-side and client-side execution environments.

Node.js is a JavaScript server-side runtime environment. Express is a web application framework that runs on top of node.js. MongoDB is a no-SQL database engine. Finally, Angular.js is a client-side framework and will be discussed near the end of this course.

We will start with a simple node.js server implementation, followed by the development of basic CRUD (Create, Update, Retrieve and Delete) operations under the express.js framework.

Hello World Server with the http Module

Node.js can be used for a wide variety of tasks, but its primary use seems to be on web application development. Node.js thrives in the network as a result of its asynchronous nature and built-in modules such as net and http.

Here』s an essential Hello World example which sends Hello World to any visitors. Create a file named as hello.js with the following code. For small program, we can use the nano terminal text editor. Type nano in terminal and paste the code below. Hit Control + x to leave the program.

var http = require('http');

http.createServer(function (req, res) {
    res.writeHead(200, {'Content-Type': 'text/plain'});
    res.end('Hello World\n');
}).listen(1337, '127.0.0.1');

console.log('Server running at http://127.0.0.1:1337/');

From the folder in which you have hello.js, launch in your terminal the following command:

node hello.js

Visit http://localhost:1337 from a browser and you should see Hello World. To shut down the server, hit Control + c.

Let』s break it down a bit. The following loads the core http module for the server:

var http = require('http');

This snippet below creates a server with a callback function which contains the response handler code.

var server = http.createServer( function (req, res) {

To set the right header and status code, use the following

res.writeHead(200, {'Content-Type': 'text/plain'});

To output Hello World with the line end symbol, use

res.end('Hello World\n');

The req and res arguments have all the information about a given HTTP request and response correspondingly.

To make the server accept requests, use the following:

listen(1337, '127.0.0.1');

Express.js

One can directly develop a web server with Node.js, but it is definitely quite complicated as functionalities such as system and path handling, url resolution and parsing, file I/O access, etc ... are not included by default. To speed up the development, we can use the handy Express.js web application framework.

To install express.js, please bring up your terminal and run

su xdeveloper

sudo npm -g install express-generator 

exit

Both sudo and su are very important and mostly used commands in Mac/Linux. sudo allow users (sudoers) to run programs with the security privileges of another user, by default the superuser. This allows the express-generator to be successfully installed in the machine. However, your student user isn't a sudoer, thus we will have to use su to switch to another account, xdeveloper, which is a sudoer created by our technical team. The exit at the end will switch back to your original account.

Please note that you don't have to run the first line in your own Mac, and the sudo is also not needed for Windows system.

npm is the package manager of Node.js. It can install a number of useful packages and modules. Here, we installed the express-generator, which can generate an "express" project with a simple command:

express --ejs myapp

This line would generate a new folder myapp with a skeleton of files automatically created inside (i.e., Scaffolding). ejs represents our chosen template engine, there're some other options like jade and handlebars.

Next, go inside this folder and install the required dependencies:

cd myapp

npm install

This simply fetches and installs all the required Node.js packages and modules, being listed in the package.json file. It takes a while. Finally, we can start the server with 

npm start

By default, the server would run on port 3000, thus we can visit this website via http://localhost:3000/. You may have to turn off the proxy in case it can't be reached.

The Routes

Next, find index.js under routes, and you should see the following around line 5 - 7.

router.get('/', function(req, res, next) {
  res.render('index', { title: 'Express' });
});

The above shows a route of our website and each route basically defines an request-response pair. Based on different requests (path and HTTP methods) from the user, we return different responses (pages and contents).

Here, get represents the GET method and / is the path. So, when the user simply visits our home page http://localhost:3000/ while no extra path is given, this matches with the /.

In this case, we will render our index.ejs, as specified in line 2. res is our response object. The second argument {title: 'Express'} is the dynamic content being passed to the index.ejs.

Don't worry, you will know more when we develop more routes. Here's the first one, append it just below the previous route.

router.get('/form', function(req, res, next) {
  res.render('form');
});

In your terminal, hit Control+C to stop the server. Hit 🔼 to show the previous command and hit RETURN to start the server again.

Then, visit http://localhost:3000/form, the form.ejs would be rendered and displayed. Also, our form is static, and thus no dynamic content is passed to it.

const MongoClient = require('mongodb').MongoClient;
const assert = require('assert');
 
// Connection URL
const url = 'mongodb://localhost:27017';
 
// Database Name
const dbName = 'myapp';

// Use connect method to connect to the server

We need the mongodb external module to communicate with the server, which was done in line 1. Line 2 loads the assert module which helps writing unit tests.

Delete

The Delete operation is quite easy to develop, we simply delete a particular document using a deleteOne() MongoDB operation.

router.get('/delete/:id', function(req, res, next) {

  db.collection('bookings').deleteOne({"_id":ObjectId(req.params.id)}, function(err, result) {
    if (err) {
      throw err;
    }

    res.send("Item removed.");
  });

});

The general idea is similar to the previous route. In line 1, we have defined the endpoint with an :id parameter, which could be accessed via req.params.id within this route.

In line 3, we use the deleteOne() mongoDB function to delete the content of a particular booking, via comparing the ObjectId ('_id') with the req.params.id. Please note that the req.params.id is a string, and we have to convert it into ObjectId before we can perform the required comparison. Thus, please add this dependency at the top of your index.js with 

var ObjectId = require('mongodb').ObjectID;

Please be reminded that allowing data to be deleted via a GET operation is not recommended. Can you make the required changes?

Update

The last operation of the CURD is discussed here. let's develop our last route.

In index.js, add the following route to display the update page, showing the current content of a particular booking.

router.get('/update/:id', function(req, res, next) {

  db.collection('bookings').findOne({"_id": ObjectId(req.params.id)}, function(err, result) {
    if (err) {
      throw err;
    }

    if (result == null) {
      res.send("no such booking");
    } else {
      res.render('update', {booking: result});
    }
  });

});

Notice the use of findOne() in line 3, which is very similar to the deleteOne() we used previously.

Basically the same as our previous form. The major change is now we filled the input elements with existing data, using <%=booking.name%> etc. Also, this form would be submitted to another handler, /update/:id, using the POST method.

So, we have to provide a route to handle this post request.

router.post('/update/:id', function(req, res, next) {

  db.collection('bookings').updateOne({"_id":ObjectId(req.params.id)},
  {
    $set:
    {
      name : req.body.name,
      email : req.body.email,
      website : req.body.website,
      numTickets : req.body.numTickets
    }
  }, function(err, booking) {
    if (err) {
      throw err;
    }

    res.send("Updated");
  });

});

In line 3, we use another function updateOne() to update existing database values. The updateOne() function accepts three input parameters: the first one is for finding/matching a particular data object, the second one is to set the new values. The last one is a callback function, which will be performed when the database operation has been completed.
